1. Complete overview of Meta events

Meta events are signals (page views, purchases, add‑to‑cart) sent from your website or server to Meta. They power ad targeting, optimization, and reporting.

🔍 Real‑life example — A user lands on your product page → ViewContent event fires. Later they buy → Purchase event. Meta attributes the purchase to the ad they clicked 2 days ago.

Two ways to send events

MethodProsCons
Browser Pixel (JavaScript)Easy, real‑time, automatic cookie matchingBlocked by ad‑blockers, ITP (Safari), 25‑40% loss
Conversions API (CAPI) (server‑side)No blockers, full control, higher match qualityRequires backend development

✅ golden rule: send every conversion event via both Pixel + CAPI with the same event_id. This gives redundancy and best data.

2. Meta Pixel (browser events)

When you drop the Pixel base code, it creates a fbq queue. Calls before library load are stored and executed in order.

Real‑world PageView in a Next.js app

// components/MetaPixel.tsx (client component)
'use client';
import { usePathname } from 'next/navigation';
import { useEffect } from 'react';

export function MetaPixel() {
  const pathname = usePathname();
  useEffect(() => {
    if (typeof window.fbq === 'function') {
      window.fbq('track', 'PageView');
    }
  }, [pathname]);
  return null;
}

What happens on the network?

Open DevTools → Network tab → filter facebook.com/tr. You'll see:

https://www.facebook.com/tr/?id=1234567890&ev=PageView&dl=...

Parameters include _fbp (browser cookie) and _fbc (click id from ad).

⚠️ ITP impact: Safari limits _fbp to 7 days, sometimes 24h. That's why we need CAPI.

📦 Node.js example with the Business SDK (auto‑hashing PII)

const bizSdk = require('facebook-nodejs-business-sdk');
const { ServerEvent, UserData, CustomData, EventRequest } = bizSdk;
const access_token = 'EAA...';
const pixel_id = '1234567890';
bizSdk.FacebookAdsApi.init(access_token);

const userData = new UserData()
  .setEmail('buyer@example.com')    // SDK hashes automatically
  .setClientIpAddress('203.0.113.5')
  .setFbp('fb.1.1710000000.12345');

const customData = new CustomData()
  .setValue(129.99)
  .setCurrency('USD')
  .setContentIds(['SKU123']);

const event = new ServerEvent()
  .setEventName('Purchase')
  .setEventTime(Math.floor(Date.now()/1000))
  .setEventId('purchase_12345')
  .setUserData(userData)
  .setCustomData(customData)
  .setActionSource('website');

const request = new EventRequest(access_token, pixel_id).setEvents([event]);
request.execute().then(console.log);

4. Deduplication (event_id is MANDATORY)

When you send from both Pixel and CAPI, you must use the same event_id to avoid double counting.

🔄 Real flow:
1. Backend generates UUID 'order_001'.
2. Frontend calls: fbq('track', 'Purchase', {...}, {eventID: 'order_001'})
3. Backend sends CAPI with "event_id": "order_001".
4. Meta merges them into one event. 
// Frontend
fbq('track', 'Purchase', {
  value: 99.99,
  currency: 'USD'
}, { eventID: 'order_001' });   // camelCase 'eventID'

// Backend CAPI payload (snake_case 'event_id')
{
  "event_name": "Purchase",
  "event_id": "order_001",       // must match
  ...
}

❗if event_id is missing: Meta counts two events → ROAS doubled, optimization broken.

6. Required vs recommended parameters

content_ids must match your product catalog’s retailer_id.

  • content_type: 'product' → IDs are SKU/variant level
  • content_type: 'product_group' → IDs are parent product (for multi‑variant)

Parameter quick reference

// CORRECT
value: 179.97, currency: 'USD'
content_ids: ["SKU123", "SKU456"]
contents: [{id:"SKU123", quantity:1, item_price:129.99}]

// WRONG
value: "179.97"            // must be number
currency: "$"              // use ISO code 'USD'
content_ids: "SKU123"      // must be array
contents: [{id:"SKU123"}]  // missing quantity & item_price

7. User data – high EMQ (event match quality)

EMQ score 0‑10. To reach 8+ you need email/phone + fbp/fbc + IP + user agent.

📧 Hashing rules (CAPI): lowercase, trim, SHA‑256. Never hash IP, user agent, fbp, fbc. 
// Node.js hashing example
const crypto = require('crypto');
function hash(value) {
  return crypto.createHash('sha256')
    .update(value.toString().toLowerCase().trim())
    .digest('hex');
}
// email: "User@Example.com" → hashed from "user@example.com"

Priority order: email ≈ phone > fbp/fbc > external_id > name > location.

8. Debugging – commands & tools

  • Pixel Helper – Chrome extension: shows events firing.
  • Events Manager (Test Events) – use test code to see real‑time events.
  • Network tab filter: facebook.com/tr – inspect payload.

Check CAPI response (always inspect)

{
  "events_received": 1,
  "messages": ["Custom data field 'value' must be a number."]   // warning but 200
}

⚠️ HTTP 200 doesn't guarantee event is valid – always read messages array.

9. Privacy, iOS 14+, AEM

Consent mode: call fbq('consent', 'revoke') before init, then fbq('consent', 'grant') after user accepts.

fbq('consent', 'revoke');
fbq('init', 'PIXEL_ID');
// later, when consent given:
fbq('consent', 'grant');  // queued events fire

iOS AEM: only 8 conversion events per domain. Prioritize Purchase, InitiateCheckout, AddToCart, etc. Must verify domain in Business Settings.

Domain verification via DNS TXT record:

# add TXT record for yourdomain.com
facebook-domain-verification=YOUR_VERIFICATION_CODE

10. Parameter Builder + Business SDK

Parameter Builder (capi-param-builder) automatically manages _fbp/_fbc cookies on your server.

Install & use (Node.js example)

npm install capi-param-builder
const { ParamBuilder } = require('capi-param-builder');
const paramBuilder = new ParamBuilder({ domainList: ['yourstore.com'] });

// middleware: process every request
app.use((req, res, next) => {
  const result = paramBuilder.processRequest({
    url: req.url,
    headers: req.headers,
    cookies: req.cookies,
    referrer: req.headers.referer
  });
  // set cookies in response
  if (result.cookies) {
    result.cookies.forEach(c => {
      res.cookie(c.name, c.value, { domain: c.domain, maxAge: c.maxAge });
    });
  }
  req.metaParams = result;
  next();
});

// later in CAPI event:
const params = paramBuilder.getEventParams(); // { fbp, fbc }
// use params.fbp, params.fbc in user_data

Combined with Business SDK (auto‑hashing + type safety)

const userData = new UserData()
  .setEmail(user.email)                     // hashed automatically
  .setFbp(req.metaParams.fbp)                // from paramBuilder
  .setFbc(req.metaParams.fbc);

✅ best practice: Parameter Builder + Business SDK = highest EMQ + less bugs.

📌 cheat sheet – do’s and don’ts

  • ✔️ Send Purchase via both Pixel & CAPI
  • ✔️ Use server‑generated event_id and share
  • ✔️ Hash PII (email, phone) with SHA‑256 after lowercasing
  • ✔️ Include _fbp and _fbc in CAPI
  • ✔️ Set action_source: 'website'
  • ❌ Never hash client_ip_address, user_agent, fbp, fbc
  • ❌ Don't send value as string
  • ❌ Don't reuse event_id for different events
